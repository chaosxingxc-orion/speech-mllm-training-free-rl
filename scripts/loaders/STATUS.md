## Reconciliation done (this task)

`registry.py` had four of six agents' sections merged (B1-asr-content, B2-paralinguistic,
B3-slu, B5-reexamined) but was **missing two entire sections** whose loader modules already
existed on disk but were never imported/registered:

- **B4-qa-mcq** — `mmar.py` (`load_mmar`) + `uro_bench.py` (16 bucket-A subset loaders via its
  `_ALL` dict) were present but not wired into `LOADERS`.
- **B6-retrieval-tools** — `audio2tool.py`, `audiocaps_qa.py`, `mmsu.py`, `squtr.py` were present
  but not wired into `LOADERS`.

No duplicate `LOADERS[...]` keys were found across the four already-merged sections (checked via
`sort | uniq -d` over every `LOADERS["..."] = ...` line — zero collisions), and no duplicate
key was introduced by adding B4/B6. All 6 loader modules' top-level imports were already
lazy-import-compliant (stdlib + `pathlib`/`re`/`functools`/`_common` only — `numpy`,
`pyarrow.parquet`, `soundfile` all deferred inside functions), so no per-loader-module fix was
needed there.

**Structural fix made**: `registry.py`'s import statements previously sat directly at module top
level (`from mmar import load_mmar`, etc.) — a syntax error or an eagerly-imported missing dep in
*any one* loader module would have raised all the way out of `import registry`, killing every
other section's registration too (this is a stronger failure mode than a loader that imports fine
but raises when *called*, which `smoke_all()` already tolerates). Rewrote every section as a
`_register_bN_*()` function called through a new `_try_register(section, fn)` wrapper that catches
`Exception`, records it in `IMPORT_ERRORS[section]`, prints a warning, and continues — so one
broken loader module can now only cost that module's own keys, never the whole registry. Verified
empirically: `IMPORT_ERRORS == {}` on this run (all 6 sections imported cleanly), and `smoke_all()`
still correctly isolates the 2 runtime (call-time) failures below without affecting the other 58.

Sections were also reordered B1→B6 (was B1, B3, B2, B5) and each section's `LOADERS[...]`
assignments alphabetized by key, purely for readability — no behavior change.

## (b) Environment blockers (2)

### `meld` — ffmpeg missing, no sudo

- **Error**: `NeedsExtraction: ffmpeg not found on PATH -- meld loader needs it to decode .mp4 ->
  .wav for 3 row(s).`
- **Root cause**: MELD test-split utterances ship as `.mp4` (video+audio); `meld.py` needs a
  decoded `.wav`. Row-building (CSV join + seeded sampling) succeeds — only the wav
  materialization step is blocked. `command -v ffmpeg` confirmed not on PATH in the WSL venv;
  this box has no sudo (CLAUDE.md), so `apt-get install ffmpeg` is not an option.
- **Exact unblock step (TODO, not run)**: install a user-space (no-sudo) static ffmpeg binary,
  e.g. one of:
  - `uv pip install imageio-ffmpeg` inside `~/.venvs/speechrl` (bundles/downloads a static
    ffmpeg binary to the pip cache at first use, no sudo) — then either add its binary dir to
    `PATH`, or point `meld.py`'s `_ffmpeg_cmd` at
    `imageio_ffmpeg.get_ffmpeg_exe()` instead of the bare `"ffmpeg"` string.
  - Download a static build (e.g. the johnvansickle.com linux-amd64 static tarball) to
    `~/.local/bin/ffmpeg` and ensure `~/.local/bin` is on `PATH`.
  - Once ffmpeg resolves on PATH, **no code change is required** — `load_meld` re-checks
    `shutil.which("ffmpeg")` on every call and `_extract_wav` materializes+caches per row
    automatically.

### `air-bench-foundation-speech-grounding` — 12/25 Foundation (task, dataset) pairs metadata-only

- **Error**: `NeedsAirBenchFoundationAudio: air-bench-foundation-speech-grounding-librispeech:
  audio dir missing -- .../datasets/air-bench/Foundation/Speech_Grounding_librispeech does not
  exist`
- **Root cause**: `docs/datasets.lock.json` records `air-bench` as `"status": "COMPLETE"`
  (40.8G, 13574 files) — but that's a **content-level false-COMPLETE**: `Foundation_meta.json`
  references audio for 25 `(task_name, dataset_name)` pairs, and only 13 of the corresponding
  `Foundation/<task>_<dataset>/` directories are actually present on disk (verified by
  `air_bench_foundation.missing_tasks()`). `Speech_Grounding_librispeech` — the one
  speech-native Foundation task, and the task this loader is asked to prioritize — is among the
  12 missing. The other 13 (Foundation-)music/sound/acoustic-scene pairs registered here all
  PASS, so this is a partial gap, not a total one.
- **Exact unblock step**: re-run the fetch for just this dataset from the umbrella repo root —
  `bash scripts/data/fetch-data.sh air-bench` (re-invokes `modelscope download --dataset
  qfq/AIR-Bench_24.09 --local_dir .../datasets/air-bench`, which resumes/fills in missing files
  rather than re-downloading the 13 already-complete pairs). After the re-fetch, re-run
  `air_bench_foundation.missing_tasks()` to confirm the gap closed, then re-smoke this key.
  Consider also filing this as a lockfile/inventory follow-up (`docs/data.md` /
  `scripts/data/inventory.sh`) since `"status": "COMPLETE"` is misleading at the
  `(task_name, dataset_name)` grain — matches the previously-documented false-COMPLETE pattern
  for shallow (top-level-only) inventory checks.

## (c) Data-layout surprises (already handled correctly by the loader; documented for visibility)

These all **PASS** — listed here only because a caller consuming their output should know the
caveat. No action needed; each is already handled in the loader's own code + docstring.

- **`air-bench-foundation-audio-grounding`** — metadata `path` values end in `.wav` but every
  on-disk file for this pair is actually `.flac` (same stem). The loader resolves by filename
  STEM against an on-disk directory index rather than joining the path literally, so this
  resolves 896/896 rows correctly; a literal-path join would silently resolve 0/896.
- **`air-bench-foundation-sound-aqa-clothoaqa`** — 1000 metadata rows reference only 351
  distinct on-disk audio clips' worth of files (multiple QA pairs can share one clip, and a
  further genuine partial-download gap exists on top of that); 400/1000 rows have no resolvable
  on-disk file and are dropped **before** sampling (never counted toward `n`, never returned as
  a dangling path) — confirmed by the printed warning during this smoke run: `400/1000 metadata
  row(s) have no resolvable on-disk audio file -- dropped before sampling`.
- **`mmsu`** — 957/5000 rows (19.1%) have no on-disk `audio/<path>.wav` despite the metadata
  parquet always carrying non-empty embedded audio bytes for every row (root cause not
  diagnosed — likely a partial earlier audio-only fetch pass predating the full metadata join).
  The loader prefers the on-disk file when present and falls back to materializing from the
  parquet's embedded bytes (cached) when it's missing, so every row is servable regardless.
- **`voicebench-ifeval`** — returns the raw `instruction_id_list`/`kwargs` gold shape but there
  is **no rule-checker implementation** bundled (no permissive-licensed reference implementation
  was locatable offline as of 2026-07-09) — a caller wanting an actual IFEval pass/fail score
  needs to bring or write that checker; this loader only supplies the verifiable-in-principle
  instruction spec, not the scoring logic (out of scope for a thin loader per the package
  contract anyway).
- **`uro-bench-TruthfulEval`** — gold is a `list[str]` of acceptable paraphrases already baked
  into the source parquet; list-containment (any member appears in the model's output) is a
  coarse proxy for the real TruthfulQA standard (a trained judge) — both under- and
  over-credits paraphrases. Treat this subset's numbers as directional only, per its own
  module-docstring note.

## Notes on what "60 registered" covers vs. what's on disk but intentionally excluded

Not every `load_*` function in `scripts/loaders/*.py` is registered — by design, per the
package's Row-list contract (see `README.md`):

- `air_bench_foundation.get_loader(task_name, dataset_name)` is a factory for the other
  `25 - 14 = 11` Foundation `(task_name, dataset_name)` pairs not hand-registered as named
  constants; only the 14 named accessors (Speech_Grounding first) are wired into `LOADERS`.
- `audio2tool.load_tools_registry()` — the flat 152-tool registry, a knowledge source (list of
  tool specs), not a per-item Row loader.
- `squtr.load_squtr_retrieval()` / `squtr.build_mini_corpus()` — the BEIR-style retrieval entry
  point returns `{"queries", "corpus_sample", "qrels"}`, a different shape than `list[Row]`; only
  `load_squtr` (one Row per query) is registered, per the module's own docstring instruction.
- `cn_celeb1.trial_pairs()` — a `(enroll, test, label)` speaker-verification trial accessor, not
  a single-item Row; documented in `registry.py`'s B2-paralinguistic section comment as
  deliberately unregistered.
- `uro_bench.py`'s `SQuAD-zh` / `OpenbookQA-zh` loaders are thin shims over the shared
  `_load_uro_subset` body re-registered here as `uro-bench-SQuAD-zh` /
  `uro-bench-OpenbookQA-zh` — the pre-existing, differently-keyed `p2_baselines.py` versions of
  the same two datasets are a separate script-local implementation predating this package and
  are out of scope for this registry.
